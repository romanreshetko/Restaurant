Программа управления заказами

1. При запуске программы требуется выбрать вход или регистрацию, после чего нужно
ввести логин и пароль пользователя. При регистрации программа запрашивает код
администратора для соответствующей регистрации ("superadmin").

2. Далее для работы нужно выбрать одну из опций из списка и следовать инструкциям далее.
Везде при вводе данных строки и числа нужно вводить без дополнительных знаков или символов.

Админ может:
    добавить новое блюдо в меню (введя название, цену, время готовки и количество);
    удалить блюдо из меню (введя название в точности, как при добавлении);
    посмотреть меню;
    посмотреть статистику (на выбор кол-во денег, среднюю оценку и отзывы блюда по названию);
    добавить количество порций блюду (введя имя и количество)
    выйти из программы.

Клиент может:
    посмотреть меню;
    создать заказ (предоставляется выбор: добавить блюдо в заказ или отправить заказ.
        При выборе блюда нужно ввести его номер в меню. После добавления всех
        нужных блюд заказ нужно отправить.)
    изменить заказ (предоставляется выбор: добавить блюдо в заказ или отменить заказ.
        После этого нужно выбрать заказ из списа последних заказов. При этом
        нельзя редактировать готовый заказ.)
    проверить статус заказа (список последних заказов с их статусами готовности)
    оставить отзыв (сначала нужно выбрать номер заказа, а потом поочерёдно
        оценить все блюда из него и оставить комментарии.
        Допустимо только для полученных заказов со статусом Ready.)
    выйти из программы.


Процесс готовки блюд симулируется многопоточностью.
Время готовки в меню указывается в минутах, но программа готовит блюдо столько секунд
    (для удобства)

При реализации программы использовались шаблоны строитель и команда
1. Так как процесс создания блюда требует нескольких взаимодействий с пользователем,
    удобнее сделать строителя, который занимается этой логикой внутри себя.

2. Команда используется для передачи заказов от клиентка на кухню.
    В данном случае заказ удобно реализовать в виде интерфейса команды,
    у которой есть метод execute(), который вызывается при готовке на кухне.
    Весь заказ передаётся в виде массива команд.
